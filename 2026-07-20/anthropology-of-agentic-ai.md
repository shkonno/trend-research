## トップ5

### 1. Anthropic Economic Index：AI利用を「職業」ではなく「タスク」の民族誌として読む
- 出典: Anthropic 公式記事 / Web
- 日付: 2025-02-10（古いが継続的に重要）
- リンク: https://www.anthropic.com/news/the-anthropic-economic-index
- 要約: AnthropicはClaudeの利用データをもとに、AIがどの職業・タスクで使われているかを分析するEconomic Indexを発表した。職業単位ではなくタスク単位で見るため、AI導入を「仕事が消える/残る」ではなく、日々の作業分解と再編として観察できる。
- なぜ面白いか:
  - 技術: 実利用ログに基づくタスク分類は、エージェント導入時の自動化候補・人間確認ポイント・権限設計を考える基礎データになる。
  - 人文: 人類学的には、これはオフィスワーカーの日課を“参与観察”する代替資料に近い。AIが労働を奪うという大きな物語より、誰が何をAIに任せ、どこで自分の判断を残すのかという職場文化の差が見えてくる。

### 2. Microsoft Work Trend Index 2025：Frontier Firmと「人間＋エージェント」組織
- 出典: Microsoft WorkLab / Web
- 日付: 2025-04-23（古いが継続的に重要）
- リンク: https://www.microsoft.com/en-us/worklab/work-trend-index/2025-the-year-the-frontier-firm-is-born
- 要約: MicrosoftのWork Trend Indexは、AIエージェントを前提にした組織を「Frontier Firm」と位置づけ、人間がエージェントを束ねる働き方へ移行すると論じている。単なる効率化ではなく、チーム編成、管理職の役割、成果物レビューの文化が変わるという文脈で読める。
- なぜ面白いか:
  - 技術: エージェントを個人ツールではなく組織の作業単位として扱うため、権限管理、監査ログ、handoff、レビュー待ちキューの設計課題が前面化する。
  - 人文: 「部下」「同僚」「外注先」のどれとしてAIを扱うのかは企業文化によって変わる。朝会・進捗報告・承認フローといった儀礼が、AI参加を前提に再発明される兆候として面白い。

### 3. OpenAI ChatGPT agent：ブラウザ操作エージェントと委任の身体感覚
- 出典: OpenAI 公式発表 / Web
- 日付: 2025-07-17（古いが継続的に重要）
- リンク: https://openai.com/index/introducing-chatgpt-agent/
- 要約: OpenAIは、ブラウザ操作や複数ステップの作業を担うChatGPT agentを発表した。人間が画面を見ながら手で操作していた作業を、AIに指示し、途中で止め、確認し、再開させるワークフローへ移すものとして捉えられる。
- なぜ面白いか:
  - 技術: UI操作型エージェントは、ツール呼び出しだけでなく、認証、画面状態、ユーザー確認、失敗時ロールバックを含む実行環境の設計を要求する。
  - 人文: 身体性の観点では、クリックやタブ移動という「手の記憶」が、プロンプト・許可・監視という別の身体技法に置き換わる。人間は作業者から見張り役へ移るが、その移行には不安、退屈、過信という感情の管理も伴う。

### 4. GitHub Copilot coding agent：IssueをAIに割り当てる開発チームの新しい儀礼
- 出典: GitHub Changelog / Web
- 日付: 2025-05-19（古いが継続的に重要）
- リンク: https://github.blog/changelog/2025-05-19-github-copilot-coding-agent-in-public-preview/
- 要約: GitHubはCopilot coding agentのパブリックプレビューを発表し、IssueをCopilotに割り当て、変更をPull Requestとして受け取る流れを示した。これはコード生成ツールというより、開発組織のチケット管理・レビュー文化にAIを入れる出来事として重要。
- なぜ面白いか:
  - 技術: エージェントがブランチを作りPRを出す形式は、既存のCI、コードレビュー、権限、監査ログにAIを接続する現実的な統合パターンになる。
  - 人文: 「誰にIssueを振るか」という開発チームの儀礼に、AIが参加者として入る。人間の熟練はコードを書くことだけでなく、AIに任せる粒度を決め、PRを疑い、チームの品質規範へ翻訳する能力へ移る。

### 5. ILO Generative AI and Jobs：職業曝露をグローバルな労働文化の差として読む
- 出典: International Labour Organization Working Paper / Web
- 日付: 2025-05-19（古いが継続的に重要）
- リンク: https://www.ilo.org/publications/generative-ai-and-jobs-refined-global-index-occupational-exposure
- 要約: ILOのWorking Paperは、生成AIへの職業曝露をタスクレベル、専門家評価、AIモデル予測を組み合わせて測る枠組みを提示している。Agentic AIそのものの論文ではないが、AIエージェントがどの労働領域に入りやすいかを、国・産業・職種の差から読む基礎資料になる。
- なぜ面白いか:
  - 技術: エージェント導入の影響評価では、モデル性能だけでなく、職務タスク、補完/代替可能性、監督コストを分けて測る必要がある。
  - 人文: 同じAIでも、国や職場の慣習によって「便利な補助者」か「監視と置換の装置」かが変わる。労働文化のローカル性を抜きに、agentic AIの普及を一枚岩で語れないことを示している。
