## 今日の一言

Claude Codeは「速く書く道具」から、権限・可観測性・アクセシビリティ・並列セッション運用まで含む“開発現場の安全な作業OS”へ寄っている。

### 1. Claude Code Week 29: MCPコネクタ付きArtifactsとスクリーンリーダーモード
- 出典: 公式ドキュメント / What's New
- 日付: 2026-07-13〜2026-07-17
- リンク: https://code.claude.com/docs/en/whats-new/2026-w29
- 要約: Week 29では、公開Artifactsが閲覧者自身のMCPコネクタを呼び出してライブデータを表示・操作できるようになり、Claude Codeにはスクリーンリーダーモードも追加された。加えて、`/fork`の背景セッション化や主要クラウド上のauto mode opt-in不要化など、日常運用に効く変更がまとまっている。
- なぜ面白いか:
  - 技術: 生成された成果物が静的なスナップショットではなく、閲覧時にMCP経由で外部状態へ接続する“生きたUI”になり始めた点が大きい。
  - 人文: アクセシビリティ対応は、AI開発ツールが一部の高速キーボードユーザーだけのものではなくなるための制度設計でもある。Artifactsの共有・編集・実行権限は、誰の接続情報で、誰の責任で、どの操作が実行されるのかという協働の倫理を前面に出している。

### 3. Claude Code v2.1.212: `/fork`、検索・サブエージェント上限、MCP自動バックグラウンド化
- 出典: 公式Changelog / GitHub release feed
- 日付: 2026-07-17
- リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.212
- 要約: v2.1.212では、`/fork`が会話を新しいバックグラウンドセッションへコピーする仕様になり、従来のセッション内サブエージェント用途は`/subtask`へ分離された。WebSearchとサブエージェント生成にはデフォルト200回のセッション上限が入り、2分を超えるMCPツール呼び出しは自動的にバックグラウンドへ回る。
- なぜ面白いか:
  - 技術: 並列化・委譲・外部ツール呼び出しを増やすほど暴走リスクが増すため、上限とバックグラウンド化がワークフロー制御の中核機能になっている。
  - 人文: “AI部下を増やす”比喩は魅力的だが、組織と同じく人数が増えるほど調整コストと責任の曖昧さも増える。今回の変更は、エージェント労働を拡張するだけでなく、止めどきと分担の境界を作る試みとして読める。

### 5. 日本語実践: CLAUDE.md・permissions・hooksを3層で分けるガードレール記事
- 出典: Qiita / 日本語実践記事
- 日付: 2026-07-18
- リンク: https://qiita.com/4q_sano/items/aaa12313849ef5999994
- 要約: 「CLAUDE.mdの禁止はお願いにすぎない」として、CLAUDE.md、permissionsのdeny、PreToolUse hookを役割分担し、実行時に決まる本番デプロイ先、現在ブランチ、`rm`対象パスなどを決定的に止めるレシピを紹介している。`bypassPermissions`や`--dangerously-skip-permissions`でも止まることを実測した、現場向けの日本語ガードレール実践として有用。
- なぜ面白いか:
  - 技術: LLMへの自然言語指示、静的deny、実行時hookを分離することで、確率的制御と決定的制御の境界を明確にしている。
  - 人文: 「禁止事項を書いたから大丈夫」という安心は、人間組織でもAI運用でも事故の温床になる。この記事は、信頼を人格や従順さではなく、検証可能な仕組みに移す文化的転換を示している。
