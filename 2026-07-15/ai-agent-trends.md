### 1. Claude Code `/checkup`: エージェント環境そのものを点検・整理する流れ
- 出典: X投稿（Boris Cherny / @bcherny）
- 日付: 2026-07-08
- リンク: https://x.com/bcherny/status/2074997570317779038
- 要約: Boris Cherny が Claude Code の新コマンド `/checkup` を紹介。未使用の skills/MCPs/plugins の整理、CLAUDE.md の重複解消、巨大なルート CLAUDE.md の分割、遅い hooks の停止、最新版への更新、auto mode や読み取りコマンド承認の調整を行う、という内容です。
- なぜ面白いか:
  - 技術: エージェント開発のボトルネックが「プロンプトを書く」から「長期運用で溜まる設定負債・文脈負債を管理する」段階へ進んだことを示しています。
  - 人文: これはAIを道具として使うだけでなく、職場の同僚や組織のように“健康診断”する発想です。人間のチームにも棚卸し・役割整理・権限見直しが必要なように、エージェントにも制度設計が必要になっています。

### 2. Claude Code / Codex 系エージェントのRCEリスク: 「読むだけ」の監査が実行事故になりうる
- 出典: X投稿（セキュリティ注意喚起として拡散）
- 日付: 2026-07-14〜2026-07-15頃
- リンク: https://x.com/connect24h/status/2077171203144593914
- 要約: X上では、未信頼リポジトリのREADME等に含まれる自然言語指示で、Claude Code や Codex の auto-review / auto-mode 系ワークフローが悪意あるバイナリを安全と誤認して実行しうる、というRCEリスクが話題になっています。推奨策として、使い捨てVM、読み取り専用マウント、外向き通信の遮断、auto mode無効化、シェルログ監視などが挙げられています。
- なぜ面白いか:
  - 技術: エージェントは外部テキストを「データ」ではなく「行動方針」として解釈しうるため、従来のコード監査よりも実行環境分離が重要になります。
  - 人文: これは“信頼”の置き場所を問い直す事件です。人間が読むREADMEと、行動できるAIが読むREADMEは社会的意味が異なり、ドキュメントが命令媒体にもなってしまう危うさがあります。

### 3. Base MCP: エージェントにオンチェーン操作の財布を持たせる公式MCP
- 出典: Web（Base公式ブログ / Base Documentation）
- 日付: 2026-05-26（古いが、7月もMCP実装例として継続的に言及）
- リンク: https://blog.base.org/base-mcp
- 要約: Base MCP は、Claude Web/Desktop/Code、ChatGPT、Codex、CursorなどのMCP対応クライアントから Base Account に接続し、残高確認、送金、スワップ、取引履歴確認、対応プロトコル利用、x402支払いをチャットから提案できる仕組みです。書き込み系アクションはユーザーがBase Account画面で確認・承認する設計になっています。
- なぜ面白いか:
  - 技術: MCPが単なる開発補助連携ではなく、金融的状態変化を伴う外部世界への標準インターフェースになりつつあることを示しています。
  - 人文: 「AIが財布を持つ」ことは、代理・責任・同意の境界を一気に具体化します。承認UIが残っている点は、人間の儀礼的な最終判断をどこに置くかという制度設計として重要です。

### 4. When Local Monitors Miss Compositional Harm: マルチエージェント安全監視の盲点
- 出典: arXiv
- 日付: 2026-07-13
- リンク: https://arxiv.org/abs/2607.11751
- 要約: 論文は、マルチエージェント・ツール利用システムで各メッセージや各ステップを個別に監視しても、複数エージェントに分割された有害ペイロードを見逃す可能性があると指摘しています。断片単体は無害に見え、組み立て後に初めて攻撃になる「distributed backdoor」を形式化しています。
- なぜ面白いか:
  - 技術: ローカル監視だけでは合成的な害を検出できないため、最終成果物・横断トレース・組み立て地点を見る監視設計が必要になります。
  - 人文: 組織犯罪や官僚制の責任分散に似て、各人・各エージェントが“無害な一部”だけを担うと全体の害が見えにくくなります。AIエージェントの倫理は個体倫理だけでなく、組織倫理として考える必要があります。

### 5. MM-ToolSandBox: 視覚付きツール呼び出しエージェントの現実的ベンチマーク
- 出典: arXiv
- 日付: 2026-07-13
- リンク: https://arxiv.org/abs/2607.11818
- 要約: Apple系の研究として、500以上のツール、16ドメイン、複数画像・複数ターン・目標変更やエラー訂正を含む、視覚的に接地したツール呼び出しエージェント評価環境を提案しています。12モデルを評価した結果、最良モデルでも成功率は50%未満で、失敗の53%は計画ではなく画像からの情報抽出ミスに起因すると報告しています。
- なぜ面白いか:
  - 技術: エージェント性能の弱点が「推論できるか」だけでなく、「画面や画像から正確に状態を読む」能力に大きく依存することを定量的に示しています。
  - 人文: 人間の仕事では、曖昧な画面・書類・写真を見て状況を読む能力が暗黙知として扱われます。この研究は、AIに仕事を任せる前に“見ること”の社会的・身体的難しさを再発見させます。
