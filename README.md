# trend-research

Hermes 自律エージェント（Docker・cron）が毎朝生成する日次トレンド調査データを保管するリポジトリです。人が手で書くのではなく、エージェントが `trend-config.json` の設定に従って自動生成します。

## 何のデータか

設定した 12 トピック（AI エージェント / Claude Code / AWS / DDD など）について、X・Web・arXiv から直近約 14 日の話題を調べ、トピックごとに「今日のトップ 5」を日本語 Markdown でまとめたものです。あわせて日次ダイジェストを 1 本作ります。

## ディレクトリ構成

日付ごとのサブフォルダに、その日の成果物をまとめています（2026-07-16 に日別レイアウトへ移行しました）。

```
trends/
├── README.md
├── templates/
│   └── topic-report.md          # トピックレポートのテンプレート
└── YYYY-MM-DD/                   # 1 日分の成果物
    ├── <topic-slug>.md          # トピック別レポート（12 本）
    ├── daily-digest.md          # その日の横断ダイジェスト
    └── daily-trends.mp3         # 音声版（※現在は生成停止。過去分のみ残存）
```

`<topic-slug>` は `aws` `claude-code` `ethics-of-ai-agents` のように、トピック名を小文字化しハイフンでつないだ形です。

## 命名規約

- トピックレポート: `YYYY-MM-DD/<slug>.md`
- 日次ダイジェスト: `YYYY-MM-DD/daily-digest.md`
- 音声（過去分のみ）: `YYYY-MM-DD/daily-trends.mp3`

## 生成のしくみ

- 構成の正本は Hermes 側の `/opt/data/trend-config.json`（`output` ブロック）です。フォルダ構成を変えるときは、リポジトリのファイルを動かすだけでは足りません。この設定と品質チェックスクリプト・cron ジョブ・スキル定義を合わせて更新してください。そうしないと翌朝の cron が旧レイアウトで書き戻します。
- トピック調査ジョブは JST 09:00〜10:17 にトピックを 1 つずつ実行し、ダイジェストは JST 10:35 に走ります。
- 音声（TTS）生成は品質の都合で現在停止しています。過去に生成した mp3 だけが各日フォルダに残っています。

## 注意

- このリポジトリは日次データを `main` に直接追加していく運用です。
- 生成物には X・Web の引用リンクが含まれます。二次利用の際は出典を確認してください。
