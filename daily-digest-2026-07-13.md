## トピック別ハイライト

### NotebookLM
- NotebookLMは、資料要約ツールから、引用付き回答、音声・動画概要、インフォグラフィック、学習カードを組み合わせる「ソース準拠の学習OS」へ広がっています。
- 教育・企業導入では、資料を読む行為が「読む・聴く・図で見る・問い直す」に分解され、知識労働の身体感覚そのものが変わりつつあります。

### Loop engineering
- 日本語記事とarXivの両方で、Loop engineeringは「プロンプト術」ではなく、停止条件、検証、状態保存、再実行を設計する運用技術として輪郭が明確になっています。
- 無限エージェントループやTest-Time Harness Evolutionの研究は、AIの自律性を高めるほど「いつ止めるか」「誰が止めるか」が中心問題になることを示しています。

### AWS
- AWS DMS Schema ConversionのMCP対応、AWS MCP ServerのOAuth/AWS Sign-In対応、KiroでのClaude Sonnet 5利用など、クラウド基盤がエージェントに操作される前提へ寄っています。
- EKSロールバックやCloudFormation Express modeは、インフラ運用をより短い検証ループに載せ、運用者の心理的安全性にも効く更新です。

### Harness engineering
- Harness engineeringは、プロンプトの外側に契約、検証、記憶、フック、監査ログを置く実装論として急速に学術化しています。
- 「From Prompts to Contracts」やProactive Memory Agentは、エージェントを信じるのではなく、追跡・検証できる制度的足場を作る方向を示します。

### sharp LLM usage
- 鋭いLLM活用の焦点は、コード生成そのものより、試行環境、テスト、敵対的レビュー、評価ループをどう組むかに移っています。
- AI生成のPR説明を禁止した事例は、流暢な文章がかえって設計意図やレビュー観点を隠す危険をよく示しています。

### AI agent trends
- Claude CodeのAuto mode、background agents、MCP roots、永続状態監査など、AIエージェントは長寿命・権限付き・監査対象の実行基盤へ進んでいます。
- 日本語圏でもCLAUDE.mdや運用ルール、ワークフロー化の記事が増え、公式機能とは別に「使い方の文化」が形成されつつあります。

### Claude Code
- Claude Codeは、CLIツールから企業設定、非対話実行、subagents背景実行、agent viewを含む開発基盤へ変化しています。
- 日本語のheadless/CI/CD実践記事とMicrosoftロールアウト研究が示す通り、導入効果は個人の速さだけでなく、予算、継続率、権限設計の問題になっています。

### Ethics of AI Agents
- CoT監視への説得攻撃、行為重大度スケール、institutional red-teamingなど、AIエージェント倫理はモデル単体より配備ルールと監査制度へ焦点が移っています。
- デジタル庁「源内」のような公共AI議論は、倫理が文化、言語、行政への信頼に埋め込まれることを思い出させます。

### Philosophy of Loop Engineering
- 「安いコード、高い判断」という構図が鮮明です。生成が容易になるほど、証拠、失敗、責任、停止判断をどうループに戻すかが哲学的・実務的な中心になります。
- 修復ループの限界効果や改ざん検知可能な実行証跡は、信頼を人格やブランドではなく、検証可能な履歴に置く流れを示しています。

### Anthropology of Agentic AI
- Agentic AIは、会議、Slack、GitHub、現場報告、ケア支援など、組織文化の記録と合意形成の仕組みに入り始めています。
- Aleenaや船舶メンテナンス報告の研究は、エージェントが作業者というより、共同体の記憶係・書記・暗黙知の翻訳者として登場していることを示します。

### History of Automation
- 自動化史の今日的論点は、雇用を奪うかどうかより、判断・学習・責任をどこに残すかです。
- 文脈アクセス格差や、AI委任による偶発的学習の喪失は、産業革命期の機械所有格差や熟練解体の現代版として読めます。

### DDD
- DDDは、AI/LLM時代に業務境界・言葉・責任をエージェントへ渡すための「社会的インターフェース」として再注目されています。
- GitHub上のDDD実践を大規模分析する研究や、AIにbounded contextを守らせる日本語実践は、コード量が増える時代ほどドメイン意図の記録が重要になることを示しています。

## 横断テーマ

### 技術テーマ
- 今日の大きな流れは、モデル性能そのものから、ループ、ハーネス、認可、記憶、監査、評価へ関心が移っていることです。
- MCP、OAuth、background agents、headless実行、実行証跡、閉ループ評価は、AIエージェントを「便利な回答者」ではなく、権限を持って働くソフトウェア部品として扱うための基盤です。
- その分、成功条件も「賢く答える」から「止まれる、検証できる、再開できる、説明できる」へ変わっています。

### 人文・社会テーマ
- エージェント時代の中心問題は、委任の文化です。誰に何を任せ、どの時点で人間が戻り、どの証拠で責任を引き受けるのかが問われています。
- NotebookLM、DDD、Anthropology of Agentic AIに共通するのは、AIが知識を処理するだけでなく、組織の記憶、会議の儀礼、現場の暗黙知、教育の身体感覚を作り替える点です。
- 自動化を「人間の置換」とだけ見ると見落とします。今日の材料はむしろ、人間が判断・規範・境界・記録を設計する役割へ移ることを示しています。
