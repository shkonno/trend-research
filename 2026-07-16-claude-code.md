### 4. AgentJailと「full auto」運用の安全柵

- 出典: X投稿（AgentJail紹介）および関連セキュリティ議論
- 日付: 2026-07-10頃
- リンク: https://x.com/AseemShrey/status/2075605097632174427
- 要約: AgentJailは、Claude Codeなどを `--dangerously-skip-permissions` で自律実行する際に、OPA/Regoポリシーでツール呼び出しを検査し、SSH鍵、.env、認証情報、危険なコマンドなどへのアクセスを制限するセキュリティラッパーとして注目されています。Claude CodeやCodexを対象にしたプロンプトインジェクション、権限昇格、RCE系の議論とも接続しています。
- なぜ面白いか:
  - 技術: エージェントの権限管理が、確認プロンプトのUX問題から、policy-as-codeで実行時に統制するインフラ問題へ移っています。
  - 人文: 「AIに任せたいが、信用しきれない」という緊張を、信頼ではなく制度設計で解く試みです。これは職場に新人を迎える時の権限設計や監査文化にも似ており、AIを同僚として扱うための社会的ルール作りに見えます。

### 5. arXiv: Claude Code実行コストとエージェント安全性の実証研究

- 出典: arXiv論文
- 日付: 2026-07-13
- リンク: https://arxiv.org/abs/2607.12161 / https://arxiv.org/abs/2607.13080 / https://arxiv.org/abs/2607.11698
- 要約: 直近のarXivでは、Claude Codeの実行ログを用いたコスト分析（「Token Reduction Is Not Cost Reduction」）、Claude CodeとオンプレLLM構成を比較する企業コーディングエージェントの推論経済学、Claude Code/Codexを対象にした自動レッドチーミング研究が確認されました。特に、単純なトークン削減が請求コスト削減に直結しない、プロンプトキャッシュや修正コミット率が実運用の評価軸になる、という点が重要です。
- なぜ面白いか:
  - 技術: Claude Codeの評価が「動いた/動かない」から、請求構造、修正負債、脆弱性再現性といった運用メトリクスへ移っています。
  - 人文: これはAIコーディングの熱狂を、経済・安全・労働の測定可能な現実へ引き戻す研究群です。エージェントがどれだけ速く書けるかだけでなく、誰が後始末し、どのコストが見えにくいまま残るのかを問う点が重要です。
